# OpenShift Virtualization: User-Defined Networking (UDN) Blog Series

## Overview
This repository contains the content strategy and scenario list for a blog series focused on **User-Defined Networking (UDN)** within OpenShift Virtualization. The series provides comprehensive guides and examples for various UDN configurations to help users move beyond the default pod network.

## Project Milestones
* **Outline Finalization**: Finalize content for Custom UDN (CUDN) and Egress.
* **Drafting Phases**: Covers primary/secondary Layer 2/3 setups and advanced connectivity scenarios.
* **Review & Publication**: Technical review and weekly publication plan.

## Testing Roadmap
Our practical lab will cover ten core scenarios:
1. **Primary Layer 2 UDN**: Testing in `udn-test1`.
2. **Primary Layer 3 UDN**: Testing in `udn-test2` with IPAM.
3. **Secondary Layer 2 UDN**: Multi-NIC setup in `udn-test1`.
4. **Services & Routes**: Validating ingress connectivity.
5. **Egress & Network Policies**: Securing traffic and external access.
6. **Localnet**: Direct VM-to-physical network attachment.
