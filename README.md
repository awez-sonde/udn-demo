# OpenShift Virtualization: User-Defined Networking (UDN) Blog Series

## Overview
This repository contains the content strategy and scenario list for a blog series focused on **User-Defined Networking (UDN)** within OpenShift Virtualization. The series provides comprehensive guides and examples for various UDN configurations to help users move beyond the default pod network.


## Testing Roadmap
Our practical lab will cover six core scenarios:
1. **Primary Layer 2 UDN**: Testing in `udn-test1`.
2. **Primary Layer 3 UDN**: Testing in `udn-test2` with IPAM.
3. **Secondary Layer 2 UDN**: Multi-homing setup in `udn-test1`.
4. **Working with CUDN**: Creating a CUDN `cluster-udn-test1` and making it available in `udn-test3`
5. **Localnet**: Direct VM-to-physical network attachment.
6. **Services & Routes**: Validating ingress connectivity.
7. **Egress & Network Policies**: Securing traffic and external access.
